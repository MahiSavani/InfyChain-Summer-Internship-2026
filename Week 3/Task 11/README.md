# Task 9: End-to-End Data Science Project — House Price Predictor (Deployment)

This builds on **Task 8** (the trained Random Forest model) by deploying it as an
interactive **Streamlit web app**, completing the full workflow:

Data collection → Data cleaning → EDA → Model building → **Deployment**

## Files
| File | Purpose |
|---|---|
| `train_and_save_model.py` | Retrains the Task 8 model on the full dataset and saves it (`model.pkl`) along with encoders and default feature values needed for deployment |
| `app.py` | The Streamlit app — user inputs house details, model predicts the sale price |
| `requirements.txt` | Python packages needed to run the app |
| `model.pkl`, `encoders.pkl`, `feature_defaults.pkl`, `feature_columns.pkl` | Saved model artifacts (generated by `train_and_save_model.py`) |

## How to run locally

1. Put `train.csv` and `test.csv` (from the House Price dataset) in this same folder.
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Train and save the model (only needs to be run once):
   ```bash
   python train_and_save_model.py
   ```
   This creates `model.pkl`, `encoders.pkl`, `feature_defaults.pkl`, `feature_columns.pkl`.
4. Launch the app:
   ```bash
   streamlit run app.py
   ```
5. Your browser will open automatically at `http://localhost:8501`.

## How the app works
- The user fills in the **most important features** (Overall Quality, Living Area,
  Basement Area, Garage Capacity, Bathrooms, Year Built, Neighborhood, etc.) through
  a simple form.
- Every other feature the model needs (out of ~79 total) is automatically filled in
  behind the scenes using the median (numeric) or most common value (categorical)
  from the training data — so the user only sees a short, simple UI, not 79 fields.
- Clicking **"Predict Sale Price"** feeds the completed feature row into the saved
  Random Forest model and displays the estimated price instantly.

## Deploying online (optional, for your video/GitHub submission)
To make it accessible via a public link instead of just `localhost`:
- **Streamlit Community Cloud** (free): push this folder to a public GitHub repo,
  then go to share.streamlit.io → connect your repo → deploy. It will auto-install
  from `requirements.txt` and run `app.py`.
- Make sure `model.pkl`, `encoders.pkl`, `feature_defaults.pkl`, and
  `feature_columns.pkl` are committed to the repo (or regenerate them by running
  `train_and_save_model.py` as a startup step) so the app has the trained model
  available.

## Result
A fully deployed, interactive house price prediction tool — user enters details,
model returns an instant prediction. This completes Task 9's required workflow and
"Simple UI" / "User input → prediction output" feature checklist.
